# 🎛 Branding Intake（質問→Brief確定→Prompt派生）

このワークフローは、製作者（あなた）にアプリごとのブランドイメージを問いかけ、
回答を **正本（brief.md）** として保存し、さらに **header_prompt.txt** を派生生成します。
以後の `/generate-header-image` / `/update-convoy-identity` は、この正本を参照して一貫性を保ちます。

## 入力（必要情報）
- productId（必須）: アプリケーションのプロダクト名。`assets/branding/{productId}/` のディレクトリ名に使用。
- アプリ名（任意）: 表示名。未指定なら `productId` を使用。

## 出力（成果物）
- `assets/branding/{productId}/brief.md`
- `assets/branding/{productId}/header_prompt.txt`
- `walkthrough.md`（本作業の質問→回答→決定事項の記録）

## 進め方（必須）
### Step 1: 質問（日本語で、短い回答でOK）
以下を順番に質問し、回答を回収する。

1. アプリ名（仮で可）と一言コンセプト（1文）
2. ターゲット（誰が／どんな場面で使うか）
3. UIの雰囲気キーワード（最大3つ）
4. 言葉遣い（例：短文・断定・手順中心 など）と **出力言語（日本語を正）**
5. NG（絶対避けたいもの：例「和風・高級・特定作品連想」）
6. 色・質感（任意：暗め/明るめ、アクセント色（１～２色）、メタル感など）
7. 参照してよい抽象モチーフ（任意：レーン/計器/グリッド等）

### Step 2: 要約（日本語）→ 1回だけ確認
- 回答を「決定事項」として箇条書きに要約し、製作者に確認する。
- 製作者が修正点を出した場合は反映して確定する。

### Step 3: brief.md を作成（正本化）
以下テンプレに落として `assets/branding/{productId}/brief.md` を作成/更新する。

```md
# Branding Brief: {productId}
```
## 1. コンセプト（1文）
{コンセプト}

## 2. ターゲット
- 主対象: {対象}
- 利用シーン: {シーン}
- 期待価値: {価値}

## 3. 想定プラットフォーム
- 対象: {Web/PWA/iOS/Android/Desktop}
- 最優先: {一つ}
- 画面前提: {モバイル優先/PC優先/レスポンシブ}

## 4. トーン&マナー（言葉遣い）
- 基調: {トーン}
- 出力言語: 日本語を正（英語は固有名詞・UI語彙のみ）
- 禁止: {NG}

## 5. ビジュアル指針（UI/画像）
- キーワード: {雰囲気キーワード}
- 色/質感: {色・質感}
- 形/情報密度: 高可読・余白重視・過度に装飾しない

## 6. モチーフ（抽象）
- OK: {抽象モチーフ}
- NG: 特定作品/固有キャラクターを想起させる表現

## 7. README/成果物の表現ルール
- 結果報告は日本語で「完了内容 / 生成・更新ファイル / 次のアクション」を含める
- 「結果は walkthrough.md を参照」の一文だけで終えない（本文に要点を要約）
